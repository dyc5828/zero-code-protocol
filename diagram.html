Capture · Orchestrate · Dispatch · Execute

/later - Capture
/next - Orchestrate
/do - Execute
/later
Inbox
Captures deferred work as TODO files for future agents.
  • Write tasks to TODO/
  • Structure task format
  • Detect worktree base
  • Ask for task details
/next
Orchestrator
The brain - analyzes, dispatches, coordinates.
  • Build dependency graph
  • Dispatch agents one at a time
  • Receive completion reports
  • Decide integration strategy
/do
Executor
The hands - executes tasks in worktrees.
  • Create worktree
  • Execute task requirements
  • Verify own work
  • Report completion
⬡
todo-store · shared storage abstraction (TODO/, tasks/, todo.md...)

Task Lifecycle Flow

1
/later captures work
User defers a task → written to TODO/ with context
↓
2
/next analyzes tasks
Reads TODO/, builds dependency graph, identifies available work
↓
3
/next dispatches first task
Picks best available task, launches agent with Task tool
↓
4
/do agents execute
Each agent: creates worktree → does work → commits → reports back
↓
5
/next receives completion
Agent finishes, reports back with structured data
↓
6
/next reevaluates and dispatches next
Archives completed task, updates graph, picks next best task (may be newly unblocked work building on predecessor)
↓
7
/next handles integration
Chooses strategy (PR-per-task, PR-per-chain, merge) based on context

Worktree Chaining

Dependent tasks build on predecessor's worktree, not main

Concurrent Execution, Sequential Decisions

Orchestrator evaluates → picks ONE task → dispatches. Agents run concurrently, but decisions are sequential and dynamic. Not batch/wave dispatch.

Intelligent Integration

Strategy chosen based on context: PR-per-task for teams, PR-per-chain for related work, direct merge for solo dev.

Multi-Agent Awareness

Agents check for resource conflicts, isolate test environments, and clean up after verification.

Self-Improvement

Skills update CLAUDE.md and memory files directly when issues are found. Larger changes captured via /later --tag meta.

Clear Handoffs

/later → TODO/, /next reads TODO/, /next dispatches /do, /do reports back, /next archives to .done/

Graceful Degradation

If permissions are blocked, agents stop and report. User is informed via orchestrator.
